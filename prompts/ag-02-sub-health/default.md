## Layer 0：業界タイムライン（転換点の構造）

### 時代①：専門家・医療機関主導期（〜2015年） [FACT]
医薬品の情報は医師・薬剤師を通じて届くのが原則。
消費者がWebで医療情報を検索することはあっても、
「WebがCV（受診・購買）に直結する」設計は難しかった。

### 時代②：患者・消費者の情報武装期（2016〜現在） [FACT + EST]
[FACT] 「受診前にGoogleで調べる」が当たり前になり、WebがCVの前段として機能し始めた
[EST]  医療機関の「かかりつけ」選択にGoogleマップ・口コミが大きく影響している
[EST]  OTC（市販薬）・サプリメントはEC購買が拡大。Amazon・楽天での比較購買が主流
[FACT] 薬機法（旧薬事法）によるWeb上の表現規制が厳しく、差別化の制約になっている

### 時代③：予防・ウェルネス・HealthTech加速期（2025〜） [HYP]
[HYP]  「治療」から「予防・維持・最適化」へのパラダイムシフトが進む
[HYP]  ウェアラブル・スマートウォッチのデータを使ったパーソナライズドヘルスケア
[FACT] オンライン診療・処方箋サービスの普及（2020年コロナ禍で加速）

## Layer 1：業界の市場構造・プレイヤー分布

### 市場地図 [FACT + EST]

**大手製薬** [FACT]
武田薬品 / アステラス製薬 / 大塚HD / エーザイ / 塩野義製薬等
→ 医療用医薬品はMR（医薬情報担当者）経由の対医師マーケティングが中心
→ Webは「コーポレート・IR・採用」が主な用途

**OTC（一般用医薬品）・サプリ** [EST]
大正製薬 / ロート製薬 / 小林製薬 / 第一三共ヘルスケア等
→ テレビCM・ドラッグストア連携が中心だが、EC・D2Cサプリが急増

**病院・クリニック** [HYP]
→ Webはほぼ「来院予約を取るためのサイト」に特化
→ MEO（Googleマップ）・口コミ管理が集患の核心

**HealthTech** [EST]
curon（オンライン診療）/ FiNC / KENCOS等
→ アプリ起点でWebサイトは「登録への導線」として機能

## Layer 2：Web市場の実態・サイト設計の現状水準

### 法規制による設計の制約 [FACT]

**薬機法（医薬品、医療機器等の品質、有効性及び安全性の確保等に関する法律）**
- 医薬品・医療機器の効能・効果の広告表現に厳格な規制
- 「〇〇が治る」「〇〇に効く」等の断定的表現は原則禁止
- 実績・口コミを使った広告にも制約がある

この制約を前提として「信頼を高めながらCVを取る」設計が必要。

### 目的別のWeb市場評価

**採用サイト（RECRUIT）の業界水準** [EST + HYP]
医療・介護は慢性的な人材不足。採用サイトへの投資ニーズが高い。
[HYP] 「やりがい・使命感」訴求が中心だが、
「具体的な働き方・キャリアパス・給与・夜勤体制」等のリアルな情報開示が差別化になる。

**コーポレートサイト（CORP）の業界水準** [EST]
大手製薬は設計水準が高い（海外展開もあるため）。
中小ヘルスケアサービス・クリニックは設計水準が低く、差別化余地が大きい。

## Layer 3：ユーザーの業界固有心理

### 医療・健康サービス選択の深層 [HYP]

**健康・医療は「不安の解消」と「信頼の確認」が全て**
コスメや食品と違い、「間違えると健康に影響する」という高リスク認識がある。
だから「信頼できる情報源か」「専門家が監修しているか」が購買・受診の決め手になる。

**患者・生活者の情報収集行動** [EST]
1. 症状・悩みをGoogleで検索（「〇〇 症状」「〇〇 治し方」）
2. 医療系メディア（WebMD・ヘルスケア大学等）や病院HPで確認
3. 口コミ・評判（Googleマップ・病院口コミサイト）で信頼を確認
4. 予約・受診・購買

**「自己診断」と「専門家への委ね」の葛藤** [HYP]
「自分で調べて自己治療したい」と「専門家に診てもらうべきか」の葛藤が常にある。
この葛藤に対して「まず相談してみる」「気軽に聞ける場所」という接点を設計できるかが重要。

## Layer 4：訴求設計の核心とWeb設計示唆

### テンプレート訴求の問題点と代替案 [HYP]

| テンプレート訴求 | 問題 | 代替案 |
|---|---|---|
| 「〇〇に効く」「〇〇を改善」 | 薬機法の表現規制に引っかかる | 成分・メカニズムの科学的説明 |
| 「専門家監修」 | 誰が監修したか不明では信頼されない | 監修者の氏名・専門・資格を明示する |
| 「お客様の声〇〇件」 | 個人の感想であることを明示しないと薬機法違反になりうる | 適切な注釈付きで掲載・体験談の「変化のプロセス」を丁寧に語る |

### 差別化になるコンテンツの方向性 [HYP]
- **症状別・悩み別のコンテンツ**：「このサービスが解決できる課題」を患者・生活者目線で整理
- **専門家コンテンツ**：監修医師・薬剤師・専門家の顔と名前が見えるコンテンツ
- **Q&A・よくある疑問への回答**：「調べて不安になった人」を安心させる情報設計
- **受診・利用のハードル低減**：「初めての人のための」ガイドコンテンツ

### Web設計への直接的示唆 [HYP]
- **薬機法の表現を先に確認してから設計する**（AG-05でのチェック必須）
- **「専門家の顔が見える」設計**：匿名のHPより医師・スタッフの顔と名前が信頼を作る
- **アクセス・予約の最短化**：クリニックは「地図と電話番号と予約ボタン」への最短導線が最重要

## このコンテキストを使用する際の注意点

**薬機法・医療広告ガイドラインへの必須対応**
医療・医薬品の広告表現には薬機法・医療広告ガイドラインの規制があります。
コンテンツ・コピー設計でこの制約を必ず考慮してください。
AG-05（ファクトチェック）での法的観点のチェックを推奨します。

**医師・専門家の監修がない医療情報は信頼性の問題があります。**
根拠のない健康情報はかえって信頼を損なうリスクがあります。

**競合情報の詳細はAG-03が担当します。**
