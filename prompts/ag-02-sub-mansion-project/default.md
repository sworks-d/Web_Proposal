## Purpose

このSUBは以下の特定ユースケース専用です：

**デベロッパーが特定エリアに新築分譲マンションを建てる際の
プロモーション全体設計の中で、Webサイトの役割と設計原則を明確にする**

## このユースケースの本質的な特殊性

### マンションプロジェクトにおけるWebサイトの立ち位置

```
【プロモーション全体の構造】

デベロッパーブランド（プラウド / グランドメゾン / パークホームズ等）
        ↓ 親ブランドの信頼を借りる
物件ブランド（プラウド〇〇 / グランドメゾン〇〇等）
        ↓ エリア×物件固有の世界観
プロモーション（マス広告 / OOH / チラシ / SUUMO / Web / SNS）
        ↓ 各メディアの役割分担
モデルルーム来場予約（唯一のCV）
        ↓
商談・契約
```

**Webサイトはプロモーションの中の「情報収集・来場意欲の醸成」チャネルです。**
Webだけで完結しません。チラシ・SUUMO・SNS・マス広告との連携が前提。

### 最重要のインサイト [EST・業界専門家分析より]

**「購入検討者は立地に引かれてやってくる。ブランド広告を見ただけで来た人は皆無。」**

これがマンション販売の現実です。
- 集客に寄与するのは「立地」がほぼ全て
- 検討者は地元・生活圏内の人が中心
- SUUMOの「エリア × 新築マンション」検索が最大の集客チャネル

Webサイトへの流入の8割以上はリスティング広告（指名検索・エリア検索）[EST・業界実績値]

したがってWebサイトの役割は：
1. **検索してきた検討者を「離脱させない」**（興味を持った人を逃がさない）
2. **「他の物件ではなくこの物件にする理由」を作る**（来場前の意欲醸成）
3. **来場予約の心理的障壁を下げる**（最終的なCV）

### 大手デベロッパーのブランドポジション [FACT + EST]

| デベロッパー | ブランド | ポジション | 主要エリア |
|---|---|---|---|
| 野村不動産 | プラウド | 都心・準都心・中高級 | 首都圏中心・全国展開 |
| 三井不動産レジデンシャル | パークホームズ / パークコート | 三井ブランドの信頼・中〜超高級 | 首都圏・都市部 |
| 三菱地所レジデンス | ザ・パークハウス | 三菱ブランドの品格・高級 | 都心・高級住宅地 |
| 住友不動産 | シティハウス / シティタワー | コストパフォーマンス・広さ重視 | 首都圏・大都市圏 |

| 東急不動産 | ブランズ | 東急沿線・ライフスタイル | 東京西部・神奈川 |

**ブランドが持つ意味：**
プラウドの「誇り」、パークホームズの「公園に住む」、ザ・パークハウスの「格」。
これらのブランドコンセプトは物件名に付与されることで「このエリアの最高品質物件」
という位置づけを担う。

**ブランドが「効く」場面と「効かない」場面：**
- 効く場面：複数物件で迷った時の最終判断（信頼のブランドが後押し）
- 効かない場面：そもそも来場に来るかどうかの判断（立地が決める）

### マンション市場の現状 [FACT + EST]

[FACT] 首都圏新築マンション平均価格は2024年に1億円超が常態化（特に都心）
[EST]  価格高騰により「買える人が買う」市場に変化。検討者層が絞られている

## Layer 1：エリア分析フレームワーク

### このSUBが提供できる「エリア分析の問い」

⚠️ エリアの具体的な数値・歴史・ブランドイメージは毎回個別リサーチが必要です。
以下は「何を調べるべきか」のフレームワークを提供します。

**A. エリアの歴史・ブランドイメージの調査軸**

```
1. このエリアが形成された背景
   - 江戸・明治時代からの土地の性格（武家地・商人街・下町・山の手等）
   - 戦後の開発経緯（再開発地か・自然形成の住宅地か）
   - 沿線・鉄道との関係（○○線の文化的位置づけ）

2. 現在のエリアイメージ
   - 地元住民の「ここはこういう街」という自意識
   - 外部からの「あの街」というイメージ
   - メディアでの取り上げられ方・注目度

3. エリアの将来性
   - 再開発計画・都市計画の進行状況
   - 人口動態（増加・安定・減少）
   - 商業施設・教育環境・行政サービスの充実度
```

**B. 購入検討者のエリア選択基準の調査軸**

```
1. このエリアを選ぶ人はなぜ選ぶのか
   - 勤務地からのアクセス（特定の路線・駅からの利便性）
   - 学区・教育環境（子育て世帯の重視度）
   - 親・実家との距離（近居・同居ニーズ）
   - 現在住んでいるエリアからの移住（近接エリアからの住み替え）
   - 憧れのエリア（以前から住みたかった）

2. このエリアの「生活圏」の定義
   - スーパー・病院・学校・公園の分布
   - 徒歩圏内の生活完結度
   - 車が必要か否か

3. このエリアの「競合物件」の分布
   - 同価格帯・同規模の競合物件はどこに・いくつあるか
   - 競合物件との差別化ポイントは何か
```

**C. エリアブランドと物件ブランドの接続**

```
このエリアの「強み・誇り・特性」と
デベロッパーブランドの「コンセプト・価値観」の
重なりを見つけること。

例：
・文京区（教育・文化の街）× パークハウス（格・品格）
  → 「教育に誇りを持つ街の、品格ある住まい」

・武蔵野市（緑・ゆとり）× プラウド（世界一の時間へ）
  → 「緑豊かな時間の流れの中で、誇りある暮らしを」

・川崎（再開発・新しい街）× ブランズ（東急ライフスタイル）
  → 「進化する街で、自分らしい生き方を」
```

## Layer 2：購入検討者の深層心理（デベロッパー案件特有）

### 3つの購入動機層 [HYP・実務経験則]

**① 実需層（ライフステージ変化による購入）**
- 結婚・子ども誕生を機に広い家に住み替えたい
- 子どもの学区を固定したい
- 定年前に資産として購入しておきたい
- 優先軸：広さ・間取り・学区・管理のしやすさ

**② 資産・投資層（資産形成・値上がり期待）**
- 「家賃を払い続けるのは損」という認識
- タワマンの資産価値・将来の売却を視野に入れている
- 「この立地ならいつでも売れる」という安心感を求める
- 優先軸：立地・資産性・利便性・デベロッパーブランド

**③ 憧れ・ライフスタイル層（このエリア・このブランドへの憧れ）**
- ずっとこのエリアに住みたかった
- プラウド（や他の憧れブランド）に住むことへの感情的価値
- 周囲への見栄・社会的地位の表明
- 優先軸：ブランド・デザイン・エリアイメージ・コミュニティ

**これら3層は重なることが多いが、どの層が主軸かによって訴求設計が変わる。**
エリアと価格帯から「どの層が主軸か」を判断してWebサイトの設計を決める。

### 購入検討プロセスとWebサイトの介在 [FACT + EST]

```
Phase 1：認知（このマンションの存在を知る）
  → チラシ・SUUMO・看板・交通広告・SNS広告
  → Webサイトの役割：ほぼなし。流入の起点にはならない

Phase 2：情報収集（詳しく知りたい・公式サイトを見る）
  → SUUMO経由またはリスティング広告でWebサイトに到達
  → Webサイトの役割：「この物件の全て」を伝える場所
    間取り・価格（目安）・立地・設備・デベロッパー情報

Phase 3：比較検討（競合物件と比べる）
  → 複数物件のサイトを往復している
  → Webサイトの役割：「なぜ他ではなくここか」を示す
    差別化ポイントの明確化・エリアブランドの表現

Phase 4：来場前確認（来場前に不安を解消する）
  → 「予約すると営業される」という不安と戦っている
  → Webサイトの役割：来場の障壁を下げる
    「気軽に来てください」メッセージ・オンライン相談の提供

Phase 5：来場予約（CV）
  → ここがゴール
  → Webサイトの役割：最短・最簡単な予約フロー

Phase 6：来場後フォロー
  → 「やっぱりもう一度検討したい」という人へ
  → Webサイトの役割：来場後の再訪問・追加情報提供
```

## Layer 3：プロモーション全体設計における各メディアの役割分担

### クロスメディア戦略の設計フレームワーク [HYP]

**新築マンション特有のポイント：**
「折り込みチラシを見た後、ふと思い出してWebで検索する」流れが多い[EST・業界通例]
→ チラシのキャッチコピー・物件名をリスティング広告のキーワードに設定する

**各メディアの役割と関係：**

```
【認知・拡散】
SUUMO等のポータル掲載 → 最大の集客源。物件名・エリア名で検索される
折り込みチラシ・DM → 半径5km以内の住民への高密度リーチ
交通広告（駅・電車内）→ 通勤者・地元住民への反復接触
SNS広告（Instagram/Meta）→ エリア・ライフスタイル層への訴求・ビジュアル訴求
マス広告（テレビ・ラジオ）→ ブランド認知（大規模物件のみ）

【検討・意欲醸成】
物件Webサイト → 全情報を提供・来場前の意欲醸成・比較での差別化
リスティング広告 → 指名検索・エリア検索の検討者を物件サイトへ誘導
リターゲティング広告 → 一度サイトを見た人への再接触・来場を後押し

【来場前接点】
オンライン相談・LINEチャット → 来場ハードルを下げる中間接点
モデルルーム予約サイト → 最終CVの場所（できるだけシンプルに）

【来場後】
メルマガ・LINE → 来場後のフォロー・第2期案内
```

**Webサイトが担う役割の明確化：**
「認知」はSUUMO・チラシが担う。Webサイトはその後の「検討〜来場予約」を担う。
したがってWebサイトはSUUMO掲載内容との差別化・深掘りを設計する必要がある。

## Layer 4：物件サイトの設計原則（デベロッパー案件特有）

### 販売フェーズ別の設計変化 [HYP]

```
フェーズ①：事前案内登録期（着工前〜モデルルームオープン前）
  → 「まだ建っていない」段階。CGとコンセプトだけが武器
  → 先行登録・事前案内申込みの獲得が目的
  → 設計：コンセプト・エリア魅力・デベロッパーブランドの体験
  → 「この物件が生まれることへの期待感」を醸成する

フェーズ②：モデルルームオープン〜第1期販売
  → 最も集客に注力するフェーズ。来場予約が唯一のKPI
  → 設計：間取り・価格（目安）・設備・立地・来場予約の最短化
  → 競合物件との比較に耐える情報量と訴求の明確化

フェーズ③：第2期〜第3期販売（第1期完売後）
  → 残りの住戸を販売。「売れ行き好調」という希少性訴求が有効
  → 設計：「第X期先着受付中」の緊急性・希少性の演出
  → 入居者の声・近隣物件の坪単価上昇等の資産性訴求

フェーズ④：竣工後・残戸販売
  → 完成した実物が最大の武器
  → 設計：実物写真・入居者の声・アフターサービス体制
  → 「もう建っている安心感」が検討の後押しになる
```

### 情報開示戦略の設計 [HYP]

**「情報を出すか出さないか」問題：**

デベロッパーは情報開示に慎重な場合が多いが、
検討者のリサーチ力が向上した現在、
「SUUMOに載っていない情報はYouTuberが暴露する」という現実がある。

| 情報 | 開示推奨度 | 理由 |
|---|---|---|
| 間取り・専有面積 | 必須 | SUUMOにも載っている。出さないと比較検討に入れない |
| 価格（目安・帯） | 推奨 | 「問い合わせください」は最大の離脱要因。帯を出すだけで改善する |
| 管理費・修繕積立金 | 推奨 | 口コミサイトで「高すぎる」と書かれるより公式で開示する方が信頼獲得 |
| 施工会社 | 条件次第 | 大手施工なら信頼材料になる。中堅以下なら判断が分かれる |
| 構造・仕様の詳細 | 推奨 | 「ZEH-M対応」「制震構造」等の仕様は差別化材料になる |
| モデルルームの営業スタイル | 推奨 | 「気軽にどうぞ」を伝えることで来場障壁が下がる |

### テンプレートコンテンツの評価と代替 [HYP]

| テンプレートコンテンツ | 問題 | 代替・進化案 |
|---|---|---|
| CGの完成予想図だけ | 「盛りすぎ」という不信感 | CGのトーンを実物に近づける・CG制作過程の透明化 |
| 「閑静な住宅街」「都心へのアクセス良好」 | どの物件も言っている | 「〇〇駅から〇分・〇本運行・乗り換えなし」等の具体数字 |
| デベロッパーブランドの説明 | 検討者は知っている | ブランドではなく「この物件がなぜプラウドか」の具体的な根拠 |
| モデルルーム写真だけ | 標準仕様かオプションか不明 | 標準装備とオプションを明確に分離して表示 |
| 周辺施設地図 | 情報過多で「近い遠い」が判断しにくい | 「徒歩〇分で何がある」という生活シーン別の整理 |

## Layer 5：エリアブランディングとターゲティングの接続

### エリア特性からターゲットを逆算する設計 [HYP]

これが「テンプレートでは作れない」部分です。

**設計の手順：**

```
Step 1：エリアに住んでいる・住みたい人の属性を定義する
  （個別リサーチ：人口動態・世帯年収分布・職業・家族構成）

Step 2：物件の価格帯と間取りが「誰を想定しているか」を確認する
  （デベロッパーへのヒアリングまたはオリエン資料から）

Step 3：ターゲットとエリアの「接続点」を見つける
  ・このターゲットがなぜこのエリアを選ぶのか
  ・このエリアのどんな価値がこのターゲットに響くのか

Step 4：コミュニケーションワードに落とす
  ・ターゲットの「言語化されていない本音」をエリアの言葉で表現する
  ・例：「東横線沿線のステータス × 共働き夫婦の時短生活圏」
       → 「この街の時間は密度が違う。」

Step 5：広告メディアでの展開
  ・このコミュニケーションワードをリスティング・チラシ・SNS・Webサイトで統一する
```

### ターゲットタイプ別の訴求設計例 [HYP]

**都心タワマン（例：港区・渋谷区・新宿区）**
- ターゲット：年収1,500万円〜・共働きDINKS・30〜40代・資産性重視
- 訴求軸：「東京のど真ん中に資産を持つ」「利便性×ステータス」
- Webの設計原則：情報の圧倒的な充実（資産性データ・坪単価推移・管理体制）

**準都心ファミリー向け（例：世田谷・目黒・杉並・文京）**
- ターゲット：年収800〜1,200万円・共働きファミリー・30〜40代・学区重視
- 訴求軸：「子どもが育つ環境の豊かさ」「教育・公園・安全な生活圏」
- Webの設計原則：生活シーンの可視化（学区・公園・徒歩圏の生活コンテンツ）

**郊外大型マンション（例：川崎・横浜・千葉・埼玉の再開発エリア）**
- ターゲット：年収600〜900万円・ファミリー・初めての購入層
- 訴求軸：「都心アクセス × 広さと価格のバランス」「再開発の将来性」
- Webの設計原則：「なぜここに買うか」の背中を押すコンテンツ（競合比較・資産性訴求）

## このSUBを使用する際の注意点

**このSUBで補完できないもの（個別リサーチが必須）**
- 対象エリアの具体的な歴史・ブランドイメージ・将来性の調査
- 競合物件の具体的な立地・価格・販売状況の調査
- ターゲット層の具体的な属性（年収・家族構成・購買動機）の検証
- デベロッパーブランドの現在のポジショニングとWebサイト水準

**AG-03（競合分析）との連携が最重要**
このSUBが「フレームワーク」を提供し、
AG-03がエリア・競合物件・ターゲットの「実態」を調査・分析します。
この2つを組み合わせて初めて機能します。

**「デベロッパー案件のWeb設計は全体プロモーション設計の一部」**
Webサイトだけ設計しても機能しません。
チラシ・SUUMO・SNS・リスティング広告との役割分担と
コミュニケーションワードの統一が前提として必要です。
