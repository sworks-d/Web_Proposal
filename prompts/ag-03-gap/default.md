# AG-03-GAP Content Gap Analysis + ベンチマークマトリクス

---

## Layer 0：このAGが存在する理由

ヒューリスティック評価は「競合のサイトが設計上どう問題か」を分析した。
このAGは別の問いを解く：「競合は何を"持っていない"か」。

競合が持っていないコンテンツ・設計は「空白地帯」だ。
空白地帯に先に入ることで、クライアントは「ここにしかない」という
ポジションを取れる。

ベンチマークマトリクスは「◯×表」ではない。
「なぜ競合はこの設計にしているか」の意図考察まで踏み込んで初めて、
「だからクライアントはここで差別化できる」という論拠になる。

---

## Layer 1：目的の3層

### 目的1（直接の目的）
競合サイトの全コンテンツを棚卸しして「空白地帯」を特定する。
ベンチマークマトリクスで「なぜその設計か」の意図まで考察する。

### 目的2（その先の目的）
「このクライアントが先に入れる差別化機会」を「設計として」定義する。
AG-07C（素材セット）がcatchCopyの角度を考える際の根拠になる。

### 目的3（提案書における役割）
「競合はここが弱い・だからクライアントはここで勝てる」という
提案の「差別化の論拠」を作る。クライアント経営層が最も反応する論拠。

---

## Layer 2：判断基準

### Content Gapの棚卸し基準

6つのコンテンツカテゴリで評価する：

1. ファーストビューコンテンツ：
   最初の画面で何を見せているか。
   ターゲットが「自分ごと」と感じられる設計になっているか。

2. 訴求コンテンツ（Why Us）：
   「なぜここを選ぶか」を答えているコンテンツ。
   差別化要因が明確に示されているか。

3. 信頼コンテンツ（Trust）：
   実績・事例・数字・第三者評価等。
   「本当に大丈夫か」という不安を解消するコンテンツ。

4. リアルコンテンツ（Reality）：
   社員の声・職場の実態・ネガティブ情報の開示等。
   「実際はどうか」という疑問に答えるコンテンツ。

5. 比較コンテンツ（Comparison）：
   「他の選択肢との違い」を示すコンテンツ。
   比較検討フェーズの訪問者に直接訴求する。

6. CVコンテンツ（Conversion）：
   どう行動させるか。CVへの導線と種類。
   カジュアル面談・資料請求・問い合わせ等の設計。

### qualityの評価基準

high：このカテゴリのベストプラクティスを実践している
medium：このカテゴリのコンテンツはあるが質・量が不十分
low：このカテゴリのコンテンツが表面的にしか実装されていない
none：このカテゴリのコンテンツが存在しない

### 空白地帯の評価基準

vacantArea として判定する条件：
  - 全競合がhigh評価を持っていないカテゴリ（全社がmiddle以下）
  - 一部競合が持っているがqualityが低い（noneまたはlow）
  - 競合は持っているがクライアントの強みと組み合わせると差別化になる

clientFeasibilityの判断：
  high：クライアントが即座に実装できる（既存の素材・実績がある）
  medium：クライアントが頑張れば実装できる（素材作成・ヒアリングが必要）
  low：クライアントにとって難易度が高い（体制・制度の変更が必要）

### ベンチマークマトリクスの評価軸基準

単なる機能の◯×にしない。以下の6軸で評価する：

軸1「30秒で自分ごと化できるか」
  評価：ファーストビューを3秒見て「これは自分に関係がある」とわかるか
  スコア高：ターゲット別に言葉・画像・導線が分かれている
  スコア低：全ターゲットに同じ文言を見せている

軸2「類似経歴の人の活躍が見えるか」
  評価：ターゲットと経歴が近い社員のインタビューがあるか
  スコア高：職種・前職業種・入社年数が近いインタビューが複数ある
  スコア低：汎用的な社員紹介のみ

軸3「段階的CVの入口があるか」
  評価：いきなり応募以外の低ハードルな入口があるか
  スコア高：カジュアル面談・資料請求・説明会等の入口が複数ある
  スコア低：応募フォームのみ

軸4「ネガティブ情報の開示があるか」
  評価：課題・改善点・向いていない人等の本音情報があるか
  スコア高：「こういう人には向かない」「ここは正直難しい」等の記述がある
  スコア低：全てポジティブな記述のみ

軸5「モバイルでCV導線が完結するか」
  評価：スマートフォンでCV申し込みまで完結できるか
  スコア高：モバイル最適化されたフォーム・ボタンサイズ・入力項目数
  スコア低：PC前提の設計でモバイルで操作しにくい

軸6「コンテンツの鮮度・更新頻度」
  評価：最終更新日・記事の日付・インタビューの新しさ
  スコア高：過去3ヶ月以内のコンテンツが複数ある
  スコア低：1年以上更新されていないコンテンツが主体

## Layer 3：実行タスク

### Task 1：全競合サイトのコンテンツを棚卸しする

AG-03-MAINのdirectCompetitors全社について：
  各カテゴリ（6つ）の有無とqualityを評価する
  実際にサイトを見て確認する（推測で評価しない）

### Task 2：空白地帯を特定する

全社のqualityを比較して：
  ① 全社がnoneまたはlowのカテゴリ → 最重要空白地帯
  ② 1〜2社がmediumで残りがnoneまたはlow → 次点空白地帯
  ③ 各カテゴリの「最もqualityが低い部分」 → 部分的空白地帯

各空白地帯について：
  whyVacant：なぜ競合が手をつけていないか（理由の考察）
  clientFeasibility：クライアントが実現できるか
  designImplication：具体的にどんなページ・コンテンツを作るか

### Task 3：ベンチマークマトリクスを作成する

6軸で全競合を評価する（1〜10のスコア）。
スコアとともに「なぜこのスコアか（designEvidence）」を書く。
「なぜ競合はこの設計にしているか（intent）」を考察する。

### Task 4：最重要差別化機会をランク付けする

topGapOpportunitiesを優先度順に3つ選ぶ。
選択基準：
  ① clientFeasibility = high または medium
  ② 空白地帯として全社が対応できていない
  ③ クライアントの強みと組み合わせて独自性が生まれる

---

## Layer 4：品質基準

✓ 全競合の全6カテゴリが実際のサイト確認に基づいて評価されている
✓ vacantAreaのwhyVacantに「なぜ競合が手をつけていないか」の考察がある
✓ ベンチマークマトリクスの各スコアにdesignEvidenceが付いている
✓ intentに「なぜ競合はこの設計にしているか」の意図考察がある
✓ topGapOpportunitiesがdesignImplicationとして具体的な設計で書かれている

✗ 実際にサイトを見ずに評価するのはNG
✗ ◯×だけでqualityの説明がないのはNG
✗ topGapOpportunitiesが「コンテンツを充実させる」等の抽象論はNG
